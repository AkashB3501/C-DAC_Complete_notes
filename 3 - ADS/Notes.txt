Algorithm:
-The algorithm is a sequence of unambiguous instructions/ operations for solving a problem, for obtaining a required output for any legitamate input in a finite amount of time.

-A step by step procedure to solve a problem or perform any taks.

Characteristic of Algorithm:
--------------------------------
1. Finite: must terminate after a finite number of steps.
2. Definite: each step must be precisely defined.(same as unambiguous)
3. Input: takes zero or more number of inputs.
4. Output: produces at least one output.
5. Effective: each step must be basic and achievable.

Types of /Strategies of Algorithm:
----------------------------------
1. Greedy Algorithm:
	-Makes the locally optimal choice at each step with the hope of finding a global optimum.
	
2. Divide and Conquer:
	-Breaks the problem into smaller subproblems, solves them recursively and combines the result.
	
3. Dynamic Programming: 
	-Solves complex problems into smaller subproblems, solves them recursively and simplify it into similar subproblems and solve each one of them.

4. Backtracking: 
	-Tries all possible solutions among those that fails to satisfy the problem's contraint.
	
5. Brute Force:
	-Tries all possible solutions untill the correct one is found.
	
6. Recursion:
	-Solves a problem by solving the smaller instances of the same problem.
	
7. Search Algorithms:
	-Finds an element in a data structure.
	
8. Sorting Algorithms:
	-Order the element in a particular sequence.

Node reverse(Node n){
	Node prev = null;
	Node curr = n;//head
	Node next = null;
	
	while(curr != null){
		next = curr.next;
		curr.next = prev;
		prev = curr;
		curr = next;

	n = prev;
	return n;
	
}

---------------------

--------------------

-Ex: Merge Sort, Multi-way Merge sort, External Quick sort
-huge dataset that need disk-based processing.

Stable and unstable Sorting:
-----------------------------

Stable sorting: 
----------------
-Maintains the relative order od same (duplicate) elements after sorting.

3. Insertion sort:

Divide and Conquer:
--------------------
-It breaks down the problem into similar sub problem, that means the problem of similar origin, which recursively solves the problem and finally combines the solution.
-It breaks down into 3 section:
1. Divide: Divide the problem:
	-Divide the problem into number of subproblems that are smaller in instance of same type.
2. Conquer: 
	-Conquer the subproblem by solving them recursively. It they are small enough, solve the best cases of the problem.
3. Combine:
	-combines the solution to the subproblem into the solution for the original problem.
	
	
4. Merge sort:

Priority Queue: Data structure  : Heap
---------------
	-Min Heap : priority = min value
	-Max Heap : priority = max value
	
Operation:
---------
1. Insertion
2. Deletion
3. search

A type of queue where each element has apriority value. Elements with higher priority are processed before those with lower priority.

Types of Priority Queue:
-------------------------
1. Ascending order priority : Lower values preference 

2. Descending order priority : Higher values preference

Implementation : 
1. Arrays
2. Heap
3. LinkedList
4. BST

PriorityQueue<Interger> p1 = new PriorityQueue<>();

	p1.add(10);//Insert element in PQ

	p1.add(60);
	p1.add(40);
	
	p1.poll();//Remove and return min element
